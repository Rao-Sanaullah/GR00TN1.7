Installation Guide

NVIDIA Isaac GR00T N1.7

WSL2 · Ubuntu 22.04 · RTX 5090 · Fine-tuning Setup

Apache 2.0 CUDA 12.8 Python 3.10 RTX 5090 · sm_120 40GB+ VRAM Recommended
⚠ Before You Begin
RTX 5090 (Blackwell / sm_120) Note The 5090 is a Blackwell GPU. Triton 3.3.1 (pinned by PyTorch 2.7) does NOT support sm_120. You will need to run a patch script after install. This guide covers it in Step 6.
CUDA Drivers Already Installed Since you have CUDA drivers on Windows, WSL2 automatically shares them. Do NOT install a separate CUDA driver inside WSL — only install the CUDA toolkit if needed.
Step-by-step Installation
1
Verify CUDA & WSL2 Prerequisites
▶

Confirm your GPU and CUDA are visible inside WSL2:

bash — WSL2 terminal
# Should show RTX 5090 and driver version
nvidia-smi

# Should return something like: release 12.8
nvcc --version
nvcc not found? Install the CUDA toolkit (not driver) inside WSL:
sudo apt install -y nvidia-cuda-toolkit
Or follow NVIDIA's CUDA on WSL guide at developer.nvidia.com/cuda-downloads

Also ensure Python 3.10 is available:

bash
python3.10 --version
# If missing:
sudo apt install -y python3.10 python3.10-venv python3.10-dev
2
Install System Dependencies
▶

Install git-lfs (required for demo data parquet files) and FFmpeg (required by torchcodec):

bash
sudo apt-get update
sudo apt-get install -y git git-lfs ffmpeg curl build-essential

# Activate git-lfs globally
git lfs install
git-lfs must be installed BEFORE cloning. If you clone first without it, the parquet files in demo_data will be missing or corrupted.
3
Clone the Repository (with submodules)
▶

Clone with --recurse-submodules to pull all external dependencies in one shot:

bash
git clone --recurse-submodules https://github.com/NVIDIA/Isaac-GR00T
cd Isaac-GR00T
Already cloned without submodules? Run: git submodule update --init --recursive
4
Install uv & Create the Environment
▶

GR00T uses uv for fast, reproducible dependency management. Install it, then create the Python 3.10 environment:

bash
# Install uv
curl -LsSf https://astral.sh/uv/install.sh | sh
# Reload shell so uv is on PATH
source $HOME/.local/bin/env

# From inside Isaac-GR00T/
uv sync --python 3.10

This installs all dependencies including flash-attn, TensorRT, and GPU extras. It will take a few minutes on first run.

Verify the install: uv run python -c "import gr00t; print('GR00T installed successfully')"
flash-attn re-validation message? Normal behavior — uv re-validates the cached wheel on each run (takes ~2–3s). Not a rebuild.
5
Fix CUDA_HOME for Fine-tuning
▶
Fine-tuning will fail with "CUDA_HOME is unset" unless you run this step.

Run the provided dependency script to configure CUDA paths:

bash
uv run bash scripts/deployment/dgpu/install_deps.sh

Or set it manually (add to your ~/.bashrc for persistence):

bash
export CUDA_HOME=/usr/local/cuda
echo 'export CUDA_HOME=/usr/local/cuda' >> ~/.bashrc
source ~/.bashrc
6
Patch Triton for RTX 5090 (Blackwell / sm_120)
▶
Critical for RTX 5090 owners. Triton 3.3.1 (pinned by PyTorch 2.7) does not recognize CUDA sm_120 (Blackwell). Without this patch you will hit a RuntimeError in ptx_get_version() during training.

Run the official patch script included in the repo:

bash
uv run bash scripts/patch_triton_cuda13.sh
Why "cuda13"? The patch script is named for CUDA 13.x platforms (Thor, Spark) but resolves the same Triton sm_ version detection bug that affects sm_120 on CUDA 12.x too. Run it regardless.
torch.compile caveat: If Triton still fails after the patch, disable torch.compile and use PyTorch eager mode for fine-tuning. TensorRT inference remains fully supported on Blackwell.
7
Prepare Your Robot Data
▶

GR00T expects data in the LeRobot v2 format with an additional meta/modality.json. Your dataset folder structure should look like:

Expected dataset layout
my_robot_dataset/
  meta/
    info.json          # dataset metadata
    episodes.jsonl     # episode index + lengths
    tasks.jsonl        # language task descriptions
    modality.json      # state/action/video key mapping ← GR00T-specific
  data/chunk-000/      # parquet files (state, action per timestep)
  videos/chunk-000/    # mp4 video files per episode
Coming from LeRobot v3? Run the conversion script: python scripts/lerobot_conversion/convert_v3_to_v2.py

Use the included SO100 demo as a reference for your own modality config:

bash
# Look at the example config structure
cat examples/SO100/so100_config.py

# Browse the 5-episode demo dataset
ls demo_data/cube_to_bowl_5/

See getting_started/finetune_new_embodiment.md for the full data prep walkthrough, including how to configure relative EEF action space for your robot.

8
Run Your First Fine-tune
▶

Single GPU fine-tune (your RTX 5090 has 32GB VRAM — meets the 40GB+ recommendation with some care; reduce batch size if you hit OOM):

bash — single GPU (RTX 5090)
CUDA_VISIBLE_DEVICES=0 uv run python \
    gr00t/experiment/launch_finetune.py \
    --base-model-path nvidia/GR00T-N1.7-3B \
    --dataset-path    demo_data/cube_to_bowl_5 \
    --embodiment-tag  NEW_EMBODIMENT \
    --modality-config-path examples/SO100/so100_config.py \
    --num-gpus        1 \
    --output-dir      /tmp/test_finetune \
    --max-steps       2000 \
    --global-batch-size 32 \
    --dataloader-num-workers 4
OOM on 32GB VRAM? Lower --global-batch-size to 16 or 8. The 40GB+ recommendation is for batch-32; smaller batches work fine with more steps.
Using your own robot data? Replace demo_data/cube_to_bowl_5 with your dataset path and examples/SO100/so100_config.py with your own modality config.

Add --use-wandb to enable Weights & Biases logging.

9
Verify with Open-Loop Evaluation
▶

After training, validate your checkpoint against ground truth from your dataset:

bash
uv run python gr00t/eval/open_loop_eval.py \
    --dataset-path  <YOUR_DATASET_PATH> \
    --embodiment-tag NEW_EMBODIMENT \
    --model-path    /tmp/test_finetune \
    --traj-ids 0 1 \
    --action-horizon 16

This saves a visualization JPEG to /tmp/open_loop_eval/traj_0.jpeg showing ground truth vs. predicted actions with MSE metrics. Use --save-plot-path <dir> to change output location.

Expect 5–6% variance between runs due to non-deterministic image augmentations — this is normal and documented in the training tips.

Training Tips for RTX 5090
→Start with --global-batch-size 16 and scale up if VRAM allows — 32GB is slightly below the 40GB recommendation but workable
→Train for 2,000–5,000 steps for new embodiments; check open-loop eval MSE to judge convergence
→--state_dropout_prob defaults to 0.2 in the fine-tune CLI. Lower it if your task relies heavily on proprioceptive state
→The new relative EEF action space in N1.7 means you should configure your robot's actions as deltas from current pose for best cross-embodiment transfer
→If torch.compile still fails after the Triton patch, add --no-compile flag or set TORCH_COMPILE=0
